How Linty Uses Your Data

When a user first signs into Linty, GitHub asks if they wish to give Linty permission to access details of their account. Linty only requests the data necessary to operate, namely, the list of repositories the user has access to and their username. When a commit is pushed to a repository watched by Linty, a copy of that commit is created inside Linty. A series of linters are then run against the code, and output is generated for reporting the linting status. Once linting is complete, the code is destroyed.

Users can disconnect their GitHub account from Linty at any time by visiting this URL. However, because Linty depends on GitHub to function, disconnected users will need to sign in again if they wish to view past results for private repositories.

Linty code is open source, so you can review it for yourself. Linty is hosted on Heroku.
